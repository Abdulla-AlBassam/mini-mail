# Mini-Mail

My final-year dissertation project: a local email system with a cascade ML spam filter.

The mail stack runs in Docker (Postfix, Dovecot, Roundcube, and a small in-network DNS) and every inbound message passes through a Postfix milter that calls a two-stage classifier. A TF-IDF and Logistic Regression baseline handles the easy cases, and DistilBERT is only invoked when the baseline is uncertain. The cascade keeps the fast model in front of the slow one without losing the slow one's accuracy on the harder messages.


## What's in this repo

- `docker/`: mail server config (Postfix, Dovecot, dnsmasq, Roundcube)
- `docker-compose.yml`: three-container stack
- `milter/`: the Postfix milter that talks to the classifier
- `ml/`: the cascade classifier and the notebooks that train both models
- `attacks/`: adversarial attack generators used to stress-test both models
- `evaluation/`: metrics and plots from clean and adversarial runs
- `evidence/`: captured TLS handshakes and SMTP/IMAP transcripts
- `scripts/`: end-to-end harnesses that send real mail through the stack

## Headline result

End-to-end run: 446/450 spam routed to Junk, 450/450 ham to Inbox across 900 messages.

## A note on the bundled keys

The TLS, CA, and DKIM private keys checked into `docker/mailserver/config/` are self-signed lab material for the fake `mail.local` domain. They exist so the stack is clone-and-run for marking and demo purposes. Do not reuse them outside this lab, and do not install the CA into any real trust store.

## How to run it

Written during development to aid writing and testing, along with reproducibility.

## Part A: Machine Learning (Spam Classifier)

### Prerequisites

- Python 3.11 or higher
- pip (comes with Python)

### Step 1: Clone the repo and enter the folder

```bash
git clone https://github.com/Abdulla-AlBassam/mini-mail.git
cd mini-mail
```

### Step 2: Create a Python virtual environment

This gives the project an isolated Python install so its libraries do not pollute (or get polluted by) anything else on the machine.

```bash
python3 -m venv venv
```

### Step 3: Activate the virtual environment

On Mac/Linux:
```bash
source venv/bin/activate
```

On Windows:
```bash
venv\Scripts\activate
```

You should see `(venv)` appear at the start of the terminal prompt.

### Step 4: Install required Python libraries

```bash
pip install -r requirements.txt
```

This installs pandas, numpy, scikit-learn, matplotlib, seaborn, joblib, nltk, ipykernel, transformers, torch, accelerate, and scipy.

### Step 5: Ensure the dataset is in place

The Enron spam dataset (49 MB) is not in git because of its size. It needs to live at:

```
ml/data/raw/enron_spam_data.csv
```

Grab the Enron-Spam CSV from Kaggle and drop it at that path.

### Step 6: Register the Jupyter kernel

This points Jupyter at the venv's Python and libraries so the notebooks can find them.

```bash
python -m ipykernel install --user --name=dissertation --display-name="Mini-Mail (Python 3.11+)"
```

### Step 7: Run the Exploratory Data Analysis notebook

```bash
jupyter notebook ml/notebooks/01_exploratory_data_analysis.ipynb
```

Pick the "Mini-Mail (Python 3.11+)" kernel when asked. Run all cells (Kernel > Restart & Run All). The notebook prints class balance, length distributions, and a couple of word-frequency charts.

### Step 8: Run the Baseline Model notebook

```bash
jupyter notebook ml/notebooks/02_baseline_model.ipynb
```

Run all cells. The notebook does the following:
1. Loads and preprocesses the dataset (33,662 emails after cleaning)
2. Splits into 80% training / 20% test
3. Builds TF-IDF features (10,000-word vocabulary with bigrams)
4. Trains a Logistic Regression classifier
5. Runs 5-fold cross-validation
6. Evaluates on the held-out test set
7. Saves charts and the trained model

**Files generated by this notebook:**

 `ml/models/baseline_logistic_regression_model.pkl` | The trained Logistic Regression model |
 `ml/models/baseline_tfidf_vectorizer.pkl` | The fitted TF-IDF vectoriser (vocabulary + weights) |
 `evaluation/baseline_confusion_matrix.png` | Confusion matrix heatmap 
 `evaluation/baseline_roc_curve.png` | ROC curve plot |
 `evaluation/baseline_feature_importance.png` | Top 20 spam/ham features chart 
 `evaluation/baseline_results.csv` | All metrics (accuracy, precision, recall, F1, ROC-AUC) |

### Step 9: Run the DistilBERT Model notebook (Google Colab)

This trains the second model (DistilBERT) on Colab's free GPU.

#### a. Upload the notebook to Google Colab

1. Go to Google Colab(https://colab.research.google.com/)
2. Click **File** then **Upload notebook**
3. Upload `ml/notebooks/03_distilbert_model.ipynb`

#### b. Enable GPU runtime

1. Click **Runtime** then **Change runtime type**
2. Set **Hardware accelerator** to **T4 GPU**
3. Click **Save**

#### c. Upload the dataset

When the notebook's upload cell prompts, upload `ml/data/raw/enron_spam_data.csv` .

#### d. Run all cells

Click **Runtime** then **Run all**. Training takes about 15 to 25 minutes on a T4 GPU.

The notebook does the following:
1. Installs HuggingFace transformers on Colab
2. Applies the same preprocessing and train/test split as the baseline
3. Tokenises with DistilBERT's WordPiece tokeniser
4. Fine-tunes `distilbert-base-uncased` for 3 epochs
5. Evaluates on the same test set with the same metrics as the baseline
6. Saves confusion matrix, ROC curve, and training-loss charts
7. Bundles everything into a zip for download

#### e. Download the outputs

The last cell triggers a browser download of `distilbert_all_outputs.zip`. Unzip it and place the files:

```
distilbert_spam_classifier/          → ml/models/distilbert_spam_classifier/
distilbert_confusion_matrix.png      → evaluation/
distilbert_roc_curve.png             → evaluation/
distilbert_training_loss.png         → evaluation/
distilbert_results.csv               → evaluation/
```

**Files generated by this notebook:**

| File | Description |
|------|-------------|
| `ml/models/distilbert_spam_classifier/` | Fine-tuned DistilBERT model and tokeniser (~260 MB) |
| `evaluation/distilbert_confusion_matrix.png` | Confusion matrix heatmap |
| `evaluation/distilbert_roc_curve.png` | ROC curve plot |
| `evaluation/distilbert_training_loss.png` | Training loss over steps |
| `evaluation/distilbert_results.csv` | All metrics (same format as baseline) |

### Step 10: Test the prediction API

From the `ml/` directory, test both classifiers:

```bash
cd ml

# Test the baseline classifier (TF-IDF + Logistic Regression)
python3 -c "
from predict import SpamClassifier
clf = SpamClassifier()

label, confidence = clf.predict('Free money', 'Click here to claim your prize now')
print(f'Baseline - Spam test: {label} (confidence: {confidence:.2%})')

label, confidence = clf.predict('Meeting tomorrow', 'Can we reschedule to 3pm?')
print(f'Baseline - Ham test: {label} (confidence: {confidence:.2%})')
"

# Test the DistilBERT classifier (only works after Step 9)
python3 -c "
from predict import DistilBertSpamClassifier
clf = DistilBertSpamClassifier()

label, confidence = clf.predict('Free money', 'Click here to claim your prize now')
print(f'DistilBERT - Spam test: {label} (confidence: {confidence:.2%})')

label, confidence = clf.predict('Meeting tomorrow', 'Can we reschedule to 3pm?')
print(f'DistilBERT - Ham test: {label} (confidence: {confidence:.2%})')
"
```

The factory function lets you switch between models without importing both classes:

```bash
python3 -c "
from predict import get_classifier
clf = get_classifier('baseline')    # or 'distilbert'
print(clf.predict('Free money', 'Click here now'))
"
```

##  Notes for users

- **Dataset not in git:** `enron_spam_data.csv` 
- **Baseline model files not in git:** The `.pkl` files in `ml/models/` are excluded. They get regenerated every time the baseline notebook (02) is run.
- **DistilBERT model files not in git:** The `ml/models/distilbert_spam_classifier/` directory (~260 MB) is excluded. It is generated by running notebook 03 on Colab and downloading the outputs.
- **Evaluation outputs not in git:** The `.png` and `.csv` files in `evaluation/` are excluded. They get regenerated by running the model notebooks.
- **Baseline results are deterministic:** All random operations use `random_state=42`. Re-running the baseline notebook gives the exact same numbers.
- **DistilBERT results are near-deterministic:** `seed=42` is set, but CUDA GPU operations are not fully deterministic. Numbers will be very close between runs but might shift by ~0.1%.
- **Virtual environment not in git:** see Steps 2 to 4 above.
- **DistilBERT dependencies:** running the DistilBERT prediction API locally needs `transformers`, `torch`, `accelerate`, and `scipy`. They are in `requirements.txt` and installed in Step 4. The Colab notebook installs its own copies separately.
